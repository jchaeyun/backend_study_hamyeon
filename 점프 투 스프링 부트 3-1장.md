### 점프 투 스프링 부트 3: 1장. 시작하기

#### 1-01 스프링 부트란?
스프링 부트(Spring Boot)는 자바 기반의 웹 프레임워크인 **스프링(Spring)**을 더 쉽고 빠르게 사용할 수 있도록 도와주는 도구이다.
* **설정의 자동화:** 복잡한 XML 설정 없이 프로젝트를 즉시 실행 가능한 상태로 만들어준다.
* **내장 서버:** 별도의 WAS(Tomcat 등) 설치 없이 단독 실행이 가능하다.
* **스타터(Starter) 의존성:** 필요한 라이브러리 묶음을 간단하게 관리할 수 있다.

#### 1-02 스프링 부트 개발 환경 준비하기
개발을 시작하기 위해 다음 세 가지 핵심 요소가 필요하다.
1.  **JDK (Java Development Kit):** 스프링 부트 3은 **Java 17 이상** 버전이 필수이다.
2.  **IDE (Integrated Development Environment):** 코드 작성을 위한 도구로 IntelliJ IDEA(권장) 또는 STS를 사용한다.
3.  **빌드 도구:** 라이브러리 관리 및 배포를 위해 **Gradle**을 주로 사용한다.

#### 1-03 스프링 부트 맛보기
스프링 부트 프로젝트의 구조와 동작 원리를 가볍게 살펴본다.
* **Controller:** 사용자의 요청을 받아 처리하고 결과를 반환하는 역할을 한다. 
* **@SpringBootApplication:** 이 애너테이션이 붙은 클래스가 프로그램의 시작점(Main)이 된다.
* 브라우저에서 `localhost:8080`에 접속하여 서버가 정상적으로 구동되는지 확인하는 과정이 핵심이다.

#### 1-04 스프링 부트 도구 설치하기
효율적인 개발을 위해 다음 도구들을 설치하고 설정한다.
1.  **IntelliJ IDEA:** 자바 개발에 최적화된 IDE로, 커뮤니티 또는 얼티밋 버전을 설치한다.
2.  **Lombok:** Getter, Setter 등 반복적인 코드를 자동으로 생성해주는 라이브러리이다.
3.  **Spring Boot DevTools:** 코드를 수정하면 서버를 자동으로 재시작해주는 편리한 기능을 제공한다.
