### **2-01. 스프링 부트 프로젝트의 구조 이해하기**
스프링 부트 프로젝트를 생성하면 표준화된 디렉터리 구조가 생성됨. 각 요소의 역할은 다음과 같음.

* **`src/main/java`**: 자바 소스 파일(`.java`)이 위치함. 컨트롤러, 서비스, 엔티티 등 백엔드 로직이 패키지 단위로 관리됨.
* **`src/main/resources`**: 자바 파일을 제외한 설정 파일 및 리소스가 위치함.
    * **`templates/`**: 타임리프(Thymeleaf) 등 템플릿 엔진 파일 보관.
    * **`static/`**: CSS, JS, 이미지 파일 보관.
    * **`application.properties`**: 프로젝트 전체 설정(DB 연결, 포트 번호 등) 기록.
* **`build.gradle`**: Gradle 빌드 도구 설정 파일. 의존성(Dependency) 및 자바 버전을 관리함.

### **2-02. 간단한 웹 프로그램 만들기**
브라우저의 요청을 처리하는 가장 기본적인 단위는 **Controller**임.

* **컨트롤러 클래스**: `@Controller` 또는 `@RestController` 어노테이션을 클래스 상단에 선언함.
* **매핑(Mapping)**: `@GetMapping("/경로")`를 통해 특정 URL 요청과 자바 메서드를 연결함.
* **응답(Response)**: 
    * `@ResponseBody`: 메서드의 리턴값을 HTTP 응답 본문에 직접 담아 보냄(단순 문자열이나 JSON 처리 시 사용).
    * 템플릿 엔진 사용 시에는 리턴값에 HTML 파일명을 적어 View를 반환함.

### **2-03. JPA로 데이터베이스 사용하기**
**JPA(Java Persistence API)**는 자바 객체와 데이터베이스 테이블을 연결하는 **ORM(Object-Relational Mapping)** 기술 표준임.

* **ORM의 특징**: 반복적인 SQL 쿼리 작성 없이, 자바 코드로 데이터를 조작할 수 있음.
* **Hibernate**: JPA 표준 사양을 실제로 구현한 프레임워크로, 스프링 부트에서 주로 사용됨.
* **H2 Database**: 개발 및 테스트 단계에서 사용하는 가벼운 인메모리 DB. 별도 설치 없이 `application.properties` 설정만으로 연동 가능함.

---

### **2-04. 엔티티(Entity)로 테이블 매핑하기**
데이터베이스의 테이블 구조를 자바 클래스로 정의한 것을 **Entity**라고 함.

* **주요 어노테이션**:
    * **`@Entity`**: 해당 클래스가 JPA 엔티티임을 명시.
    * **`@Id`**: 테이블의 기본 키(Primary Key) 설정.
    * **`@GeneratedValue`**: 기본 키 값을 자동으로 생성(Identity 전략 등).
    * **`@Column`**: 컬럼 속성(길이, 널 허용 여부 등) 정의.
* **Lombok 활용**: `@Getter`, `@Setter`를 통해 반복적인 코드를 줄임.

---

### **2-05. 리포지토리(Repository)로 데이터베이스 관리하기**
엔티티에 의해 생성된 DB 테이블에 접근하여 CRUD(Create, Read, Update, Delete)를 수행하는 인터페이스임.

* **JpaRepository 상속**: `JpaRepository<Entity클래스, ID타입>`을 상속받는 인터페이스를 생성함.
* **제공 메서드**:
    * `save()`: 데이터 저장 및 수정.
    * `findAll()`: 전체 데이터 조회.
    * `findById()`: 특정 ID로 데이터 조회.
* **쿼리 메서드**: 메서드 이름 규칙에 따라 `findByTitle`과 같이 선언하면 JPA가 자동으로 SQL 쿼리를 생성하여 실행함.
